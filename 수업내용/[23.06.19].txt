[ 가상함수의 동작원리 ]
- 가상함수를 선언하는 순간 가상함수 테이블이 생성된다.
- 실제 가상함수는 외부 메모리 어딘가에 위치하게 되고 가상 함수 테이블의 시작 주소가 클래스 객체에 추가된다. 때문에 메모리 객체의 크기가 예상보다 커진다.
- 부모 객체로 타입 변경해서 받아줘도 가상함수 테이블에 의해서 자식 객체로 찾아간다.
- virtual 키워드를 제거해주면 가상함수 테이블이 생성되지 않기 때문에 객체의 메모리 크기가 늘어나지 않는다.
- 가상함수를 사용할 때 부모 객체의 소멸차에도 virtual 키워드를 붙여줘야 제대로 메모리 해제가 된다. 그렇지 않으면 자식 객체의 메모리를 해제할 때 부모의 소멸자를 호출하게 된다.
- 윤성우 책 보면 자세히 설명되어 있다.
- 활용?

[순수 가상함수]
- 함수에 '= 0'을 붙인다.
- 순수가상함수를 붙이면 '무조건' 오버라이드 해줘야한다.
ex)
	virtual void Howling() = 0;
- 순수 가상함수가 하나라도 들어있게 되면 해당 클래스는 추상클래스가 된다.
- 추상클래스는 객체 생성이 불가능하다. 무조건 상속받아서 사용해야한다.
- 추상클래스로 객체를 만들려고 하면 에러가 발생한다.
